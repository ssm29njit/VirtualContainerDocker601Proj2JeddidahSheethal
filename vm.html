Virtualization is all about creating a virtual, instead of actual, version of computer hardware platforms, storage devices, and computer network resources in the computing world. It requires division of hardware partitions and distributing among the virtual machines present. This is a resource intensive process. This method requires logical division of system resources provided by mainframe computers between applications. .

Hardware virtualization or platform virtualization refers to the creation of a virtual machine that acts like a real computer with operating system. Hardware resources are separated from software executed on these virtual machines. An example would be, a computer might have Microsoft Windows base OS on top of underlying hardware, and the same computer may virtually host Ubuntu Linux operating system, where Ubuntu software can be run on the virtual machine.

There are two types of virtualization. One, full virtualization; it is a complete simulation of the actual hardware to allow software environments, including guest operating system and its apps, to run unmodified. Paravirtualization refers to a situation here guest apps are executed in their own isolated domains, as if they are running on a separate system, but a hardware environment is not simulated. Guest programs are required to be modified with specifics to run in this environment. .

The main common goal of virtualization is to centralize administrative tasks while improving scalability and overall hardware-resource utilization. Parallelization of several operating systems on a single CPU is the main benefit of Virtualization. Parallelism helps reduce overhead costs and differs from multitasking, involving running of several programs on the same OS. With Virtualization, enterprises can better manage updates and rapid changes to the operating system and applications without disruption to the user.

Saved state of virtual machine is called a snapshot, of an exact point of storage. A snapshot enables the virtual machineâ€™s state at the time of the snapshot to be restored later, effectively undoing any changes that occurred afterwards. This is essentially a backup technique, for example, prior to performing a risky operation. Snapshot benefits include the ability to port it over to another host machine with its own hypervisor. .

If VM temporarily stops, one can snapshot, move to another host, and then resume on the new host. This method is called migration. On the other hand, failover,allows VM to continue operations if the host fails. Generally it occurs if the migration has stopped working. .

Nested Virtualization enables running virtual machine within another VM. As a concept, it is extendable to an arbitrary depth. It allows running one or multiple hypervisors inside another hypervisor. The way nested virtualization can be implemented on a particular computer architecture depends on supported hardware-assisted virtualization capabilities. In case a device does not support hardware resources to enable nested virtualization, software techniques are required to enable it.

Virtual machines running proprietary operating systems require licensing, regardless of the host machine's operating system.
